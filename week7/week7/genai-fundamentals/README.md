# GenAI Fundamentals — Concepts + Prompt Engineering Practice

## 1. What is Generative AI?

Traditional ("discriminative") AI classifies or predicts from existing
data — spam/not-spam, cat/dog. **Generative AI** creates new content —
text, code, images — by learning the patterns of huge datasets and
sampling from them. GPT models, GitHub Copilot, and Claude are all
generative.

**Evolution (just know the timeline):**
Early chatbots (1960s) → GANs (2014) → GPT-3 (2020) → ChatGPT (2022) →
GitHub Copilot and coding assistants (now)

## 2. Prompt Engineering — the three core techniques

### Zero-shot prompting
Ask directly, no examples given. Works for simple, well-defined tasks.
```
Write a C# method that reverses a string.
```

### Few-shot prompting
Give 1-3 examples first so the model matches your expected format/style.
```
Convert these to camelCase:
student_name -> studentName
course_id -> courseId
enrollment_date -> ???
```
This is useful when you need consistent output formatting (e.g. always
JSON, always a specific naming convention) that zero-shot won't reliably give you.

### Chain-of-thought prompting
Ask the model to reason step by step before answering — improves accuracy
on multi-step logic/math problems.
```
A student has 5 courses, each worth different credits: 4, 3, 4, 2, 3.
They need a minimum of 15 credits to be full-time. Are they full-time?
Think step by step before answering.
```

## 3. Hands-on: write your own prompts (do this for real)

Try these in Claude, ChatGPT, or GitHub Copilot Chat and compare results:

**Exercise 1 (zero-shot):**
> "Write a C# method that validates whether a string is a valid email address."

**Exercise 2 (few-shot):**
> "Here are two examples of good git commit messages:
> - 'fix: resolve null reference in CourseService.GetById'
> - 'feat: add JWT refresh token endpoint'
> Write a commit message for: I added pagination to the course listing API."

**Exercise 3 (chain-of-thought):**
> "I have a React app with 3 components: App, CourseList, CourseCard.
> App fetches data and needs to pass it down to CourseCard through
> CourseList. Walk through, step by step, how props would flow, then
> write the code."

Compare: did chain-of-thought actually produce cleaner reasoning/code
than just asking directly? That comparison IS the learning objective —
document 2-3 sentences on what you noticed for your own notes.

## 4. Best practices
- Give clear, specific instructions (not "make this better" — say what "better" means)
- Provide context (paste the actual error, the actual code, the actual constraint)
- Specify the output format explicitly ("respond with only the code, no explanation")
- Iterate — first response is a draft, refine with follow-ups

## 5. GitHub Copilot — what it is and how to set it up

1. Install the **GitHub Copilot** extension in VS Code
2. Sign in with your GitHub account (Copilot requires a subscription or
   the free tier/student plan — check eligibility via GitHub Education
   if you have a student email)
3. Try it: open any file, type a comment describing what you want:
   ```csharp
   // method that calculates GPA from a list of (grade, credits) pairs
   ```
   and press Tab to accept the suggestion Copilot generates below it.

**Core features to try:**
- Tab-complete suggestions as you type
- Generate a function from a comment (like above)
- Ask Copilot Chat to explain a chunk of unfamiliar code
- Ask it to write unit tests for a method you just wrote

## 6. Risks and responsible use (know these for the quiz)
- **Hallucinations** — AI can confidently generate code that looks right but is wrong or uses APIs that don't exist. Always verify against docs.
- **Licensing/copyleft risk** — Copilot can occasionally reproduce snippets close to training data; be cautious with anything that looks suspiciously specific/verbatim.
- **Data privacy** — know what your code-sharing settings send to GitHub's servers, especially for proprietary/company code.
- **Don't blindly trust security-sensitive code** (auth, crypto, input validation) generated by AI — review it as carefully as you would a junior dev's PR.
